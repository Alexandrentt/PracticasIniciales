**2. Diagnósticos**

**Introducción**

El diagnóstico es un **procedimiento ordenado y sistemático** cuyo
objetivo es conocer y establecer con claridad una circunstancia
específica a partir de observaciones y datos concretos. En términos
institucionales, se define como un método de análisis del desempeño que
facilita la **toma de decisiones** estratégicas. Para que sea efectivo,
requiere que el analista posea razonamiento lógico, objetividad y la
capacidad de relacionar datos diversos.

- **Ejemplo:** Una **auditoría de ciberseguridad** en la que un experto
  analiza los registros de acceso y las configuraciones de un
  cortafuegos para identificar por qué se están produciendo bloqueos
  inesperados de IP, como los generados por servicios de protección tipo
  Cloudflare.

**2.1 Tipos e instrumentos**

Es vital distinguir entre la técnica (el procedimiento o \"cómo\"), el
instrumento (el soporte o \"con qué\") y la herramienta (el recurso,
usualmente digital, o \"dónde\"):

- **Técnicas:** Procedimientos como la observación, encuestas o análisis
  de redes de comunicación.

- **Instrumentos:** Soportes para registrar información, como rúbricas,
  listas de cotejo o cuestionarios especializados.

- **Herramientas:** Plataformas que facilitan la aplicación, como
  formularios online o software de análisis.

- **Ejemplo:** Un investigador utiliza la **técnica de minería de
  datos** (procedimiento) mediante un **script de Python** (instrumento)
  ejecutado en un **entorno de computación en la nube** (herramienta)
  para organizar grandes volúmenes de información.

**2.1.2 De línea, de base y basal**

Este diagnóstico se realiza para determinar la **situación actual y las
tendencias** de un sistema o proceso. Se basa en hechos recogidos
sistemáticamente que sirven como punto de partida para juzgar qué está
ocurriendo y es la base imprescindible para cualquier **planificación
integral**.

- **Ejemplo:** Antes de migrar un sistema a una nueva arquitectura, se
  realiza un **test de rendimiento basal** para establecer los tiempos
  de respuesta actuales del servidor. Esto servirá como línea de base
  para comparar si el nuevo sistema es realmente más eficiente.

**2.1.2 De potencialidades**

Se refiere al proceso de identificar y valorar las **capacidades,
habilidades y aptitudes** de un individuo para desarrollar tareas
complejas o asumir roles futuros. Utiliza evaluaciones psicométricas,
pruebas de habilidades cognitivas y análisis de competencias técnicas
para medir el potencial de crecimiento.

**Instrumentos recomendados:**

- **Pruebas Psicométricas:** Evaluaciones estandarizadas que miden
  habilidades cognitivas, rasgos de personalidad y estilos de trabajo.

- **Centros de Evaluación (Assessment Centers):** Simulaciones de
  situaciones laborales reales donde se observa empíricamente cómo
  reacciona el candidato.

- **Evaluación de 360 grados y Retroalimentación:** Recolección de
  percepciones no solo de los superiores, sino de pares y subordinados,
  para tener una visión integral del potencial.

> **Ejemplo:** En un centro de investigación, se evalúa el **potencial
> de un asistente de laboratorio** para convertirse en investigador
> principal mediante pruebas de **razonamiento lógico**, simulaciones de
> resolución de problemas experimentales y la evaluación de su capacidad
> para redactar artículos científicos.

**2.1.3 De FODA (o DOFA)**

Es una herramienta de planificación estratégica que analiza los factores
clave para el éxito dividiéndolos en dos dimensiones:

1.  **Análisis Interno (Fortalezas y Debilidades):** Aspectos propios
    del proyecto o empresa sobre los que se tiene **control directo**.

2.  **Análisis Externo (Oportunidades y Amenazas):** Factores del
    entorno sobre los que se tiene **poco o ningún control**, pero que
    impactan el éxito.

> **Instrumentos recomendados:**

- **Lluvia de ideas (Brainstorming) dirigida:** Sesiones colaborativas
  con directivos y actores clave (stakeholders) para identificar de
  forma conjunta los cuatro cuadrantes.

- **Análisis PESTEL:** Un instrumento complementario para nutrir el lado
  \"externo\" del FODA, evaluando factores Políticos, Económicos,
  Sociales, Tecnológicos, Ecológicos y Legales.

- **Matriz de Impacto:** Herramienta de ponderación para calificar qué
  amenaza o debilidad requiere atención inmediata frente a cuáles pueden
  tolerarse a corto plazo.

> **Ejemplo:** Un equipo de desarrollo de software analiza su nuevo
> proyecto:

- **Fortaleza:** Uso de una base de datos de alta velocidad (control
  interno).

- **Debilidad:** Documentación técnica incompleta (control interno).

- **Oportunidad:** Creciente demanda de aplicaciones de IA en el mercado
  (entorno).

- **Amenaza:** Nuevas regulaciones internacionales de protección de
  datos que podrían restringir el flujo de información (entorno).

Tomando como referencia un contexto **informático y de ciberseguridad**,
un ejemplo práctico de un análisis **FODA (o DOFA)** se estructuraría de
la siguiente manera:

El análisis FODA se divide en dos dimensiones: la **interna**, que
incluye fortalezas y debilidades sobre las que la organización tiene
control directo, y la **externa**, que analiza oportunidades y amenazas
del entorno donde el control es escaso o nulo.

**Ejemplo: Lanzamiento de una Plataforma de Seguridad Informática**

**Fortalezas (Internas - Control Directo):\
**

- Contar con un equipo de desarrollo altamente capacitado con títulos de
  **Grado en Ciberseguridad** y **Máster en Inteligencia Artificial**.

- Poseer una infraestructura tecnológica avanzada que permite el
  **análisis de datos masivos (Big Data)** para detectar patrones de
  ataque en tiempo real.

**Debilidades (Internas - Control Directo):\
**

- Presupuesto limitado para campañas de marketing a gran escala, lo que
  dificulta competir con grandes corporaciones.

- Falta de procesos formalizados para la **documentación técnica** de
  las redes de comunicación interna.

**Oportunidades (Externas - Poco/Sin Control):**

- Creciente demanda en el mercado por servicios de protección contra
  **ataques en línea**, inyecciones SQL y manejo de datos malformados.

- Tendencia al alza en la adopción de normativas internacionales que
  obligan a las empresas a realizar **diagnósticos funcionales** de sus
  sistemas de comunicación.

**Amenazas (Externas - Poco/Sin Control):\
**

- Surgimiento de nuevos tipos de **ataques cibernéticos** que
  evolucionan más rápido que las soluciones de seguridad actuales.

- Políticas de seguridad de terceros (como servicios de protección tipo
  **Cloudflare**) que podrían bloquear por error el acceso legítimo de
  usuarios a la plataforma debido a configuraciones de IP o cookies.

El objetivo de este análisis es resaltar las **ventajas diferenciales**
frente a la competencia de manera realista y objetiva para guiar la toma
de decisiones estratégicas.

Técnico (Palabras Clave)**

- **Análisis PESTEL:** Instrumento de evaluación externa complementario
  al FODA que analiza factores Políticos, Económicos, Sociales,
  Tecnológicos, Ecológicos y Legales del entorno.

- **Assessment Centers (Centros de Evaluación):** Instrumento de
  diagnóstico de potencialidades basado en la simulación empírica de
  situaciones laborales reales para observar la reacción de un
  candidato.

- **Diagnóstico de Línea Base:** Evaluación inicial que recoge datos
  sistemáticos para establecer la situación actual de un sistema y
  servir como punto de comparación futuro.

- **Evaluación 360 grados:** Recolección de percepciones y
  retroalimentación que involucra a superiores, pares y subordinados
  para obtener una visión integral del desempeño o potencial de un
  individuo.

- **FODA (Fortalezas, Oportunidades, Debilidades, Amenazas):**
  Herramienta de planificación estratégica que evalúa los factores de
  éxito internos (controlables) y externos (no controlables) de un
  proyecto o empresa.

- **Herramienta:** En el contexto del diagnóstico, es el recurso o
  plataforma (usualmente digital) donde se ejecuta o aplica la
  evaluación (ej. entorno en la nube).

- **Instrumento:** El soporte estructurado utilizado para registrar o
  procesar la información del diagnóstico (ej. script de programación,
  cuestionario).

- **Técnica:** El procedimiento metodológico conceptual utilizado para
  recolectar información (ej. minería de datos, observación).

**Evaluación Técnica**

**Instrucciones:** Selecciona la respuesta correcta para cada una de las
siguientes situaciones basadas en la teoría de diagnósticos.

**1. Un equipo de ciberseguridad decide utilizar un procedimiento de
\"Análisis de tráfico de red\" utilizando un \"Script en Bash\"
ejecutado dentro de una \"Máquina Virtual en AWS\". Según los conceptos
del texto, ¿qué representa el \"Script en Bash\"?** a) La técnica b) La
herramienta c) El instrumento d) La línea base

**2. ¿Cuál es el propósito principal de realizar un \"Diagnóstico de
Línea Base\" antes de migrar una base de datos a un nuevo proveedor?**
a) Evaluar las fortalezas y debilidades del equipo de programadores. b)
Establecer los tiempos de respuesta y estado actual para poder
compararlos con el rendimiento del nuevo sistema. c) Identificar las
amenazas legales (PESTEL) que existen en el entorno tecnológico. d)
Medir mediante pruebas psicométricas si el administrador de bases de
datos puede asumir un puesto directivo.

**3. Durante la creación de una matriz FODA para el lanzamiento de un
nuevo firewall corporativo, se identifica que \"han surgido nuevas
normativas internacionales de protección de datos\". ¿En qué cuadrante
debe ubicarse este factor y por qué?** a) Oportunidades o Amenazas,
porque es un factor del entorno sobre el cual la empresa tiene poco o
ningún control. b) Fortalezas, porque obliga a la empresa a mejorar su
producto de forma interna. c) Debilidades, porque es una deficiencia
técnica del equipo de desarrollo. d) Fortalezas o Debilidades, porque
depende del control directo que tenga el analista sobre la ley.

**4. ¿Qué instrumento es el MÁS adecuado si se desea aplicar un
Diagnóstico de Potencialidades para saber cómo reaccionará un analista
SOC (Centro de Operaciones de Seguridad) ante un ciberataque en tiempo
real?** a) Análisis PESTEL b) Lluvia de ideas (Brainstorming) c) Rúbrica
de desempeño histórico d) Centros de Evaluación (Assessment Centers)

------------------------------------------------------------------------

*Respuestas correctas para el docente/evaluador:* *1: c (El script es el
soporte con el que se aplica el análisis).* *2: b (Sirve como punto de
partida/comparación).* *3: a (Normativas internacionales son variables
externas/entorno).* *4: d (Permiten simulaciones de situaciones
laborales reales).*
